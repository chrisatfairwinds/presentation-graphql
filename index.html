GraphQL

What?

An RPC framework

Made by Facebook in 2015

Basics

  • Single endpoint (POST)
  • Aggregates “leaf nodes”
    • SQL
    • NoSQL
    • External APIs
  • Unique Query Syntax

Features: Type System

strings, numbers, and IDs

type Query {
  currentUser: User
}

type User {
  id: ID!
  name: String!
}

Features: Queries

query CurrentUser {
  currentUser {
    name
  }
}

Features: Mutations

mutation CreateUser($name: String!, $age: Int!) {
  createUser(userName: $name, age: $age) {
    name
    age
  }
}

Features: Subscriptions

In spec but Client/Server implementation specific

subscription {
  newPerson {
    name
    age
  }
}

Why?

Data over the wire

Select only fields you want

odata has had this for a long time but odata is very slow and relies on the url which has a max length

Amount of requests

Aggregate requests into 1

Does HTTP2 solve this?

Why not?

  • N+1 queries
  • Caching gets tricky

Caching gets handled by clients/servers like Apollo instead of the Browser
