Feature Flagging

What?

Popular Usage

Used in many popular apps and websites.

  • Web Browsers (chrome://flags)
  • Twitch
  • Github (Experimental) and https://github.blog/2021-04-27-ship-code-faster-safer-feature-flags/
  • Twitter
  • Facebook
  • YouTube

Use-cases

  • UI Changes (layout, verbiage/copy)
  • Payment Providers (enabling/disabling)
  • Gradual Rollouts (rolling out to portions of your user base at a time)
  • Single source of truth for pricing changes

Why Remote Flags?

Time

Deployment to production often takes a long time.

Time: Best Case - Continuos Delivery

Projects can take upwards of 20 to 30 minutes to build and test. (not uncommon)

PER ENVIRONMENT

envs: - dev: PR review and merging + build - staging: build + QA - production

60 to 90 minutes for a release

Time: Worst Case - Mobile Apps

Getting an update deployed can easily take a week

  • Review Process (1 Day commonly)
  • Possible Changes

Vexilla (Cons)

  • does not currently consider value based flags (pricing, etc). only cares about boolean “should”
  • no realtime listening (polling is typically fine if its just a json file)
  • ugly config UI (its on the TODO board)
  • Very Unopinionated
